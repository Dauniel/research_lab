CELLPOSE 3 vs OLD CELLPOSE — KEY DIFFERENCES
  Three concrete changes account for the PC improvement (2.04 → 3.33):

  1. Denoising (new)
     Cellpose 3 runs DenoiseModel cyto3 on every slice before
     segmentation, suppressing shot noise and sharpening condensate
     boundaries. The old pipeline segments the raw image directly
     with no preprocessing. Note: PC intensities are always computed
     from the original raw pixels — denoising only guides boundary
     placement, not the intensity values used in the ratio.

  2. 3D mode (new)
     Cellpose 3 uses do_3D=True, processing XY, XZ, and YZ planes
     simultaneously and merging gradient flows across all three before
     drawing boundaries. The old pipeline runs 2D segmentation
     independently on each of the 55 Z-slices with no cross-slice
     awareness. A condensate spanning 3 slices gets segmented 3
     separate times, often with inconsistent boundaries per slice —
     directly hurting the intensity ratio.

  3. Model version: cyto3 vs cyto2
     cyto3 is trained on a larger and more diverse dataset than cyto2,
     and is generally better at small, dim, or irregularly shaped
     objects. The 3D mode is likely the largest single contributor
     to the PC improvement.

PC COMPARABILITY NOTE
  The PC formula is identical across all models:
    cond_pixels   = Ch2[ cond_mask & nuc_mask ]   (inside condensate AND nucleus)
    dilute_pixels = Ch2[ nuc_mask & ~cond_mask ]  (inside nucleus, outside condensate)
    PC = mean(cond_pixels) / mean(dilute_pixels)

  However, the masks feeding into this formula differ:

  Model            Condensate mask         Nuclei mask
  ---------------  ----------------------  ----------------------
  Old Cellpose     Cellpose cyto2 (2D)     Cellpose cyto2 (2D)
  Cellpose 3       Cellpose cyto3 (3D)     Cellpose cyto3 (3D)
  StarDist         StarDist2D              StarDist2D
  U-FISH           U-FISH spot + disk      Otsu threshold (!)
  Nellie           Frangi + graph          Frangi + graph

  U-FISH is the outlier — it uses a simple Otsu threshold with
  morphological closing for the nuclei mask rather than a trained
  model. This gives it a different nuclear boundary than the others,
  directly affecting which pixels count as dilute-phase and therefore
  the PC. Its high PC (3.81) should be interpreted with this caveat.

  For a truly fair comparison, all models should use the same nuclear
  mask (e.g. Cellpose 3 nuclei masks) and only swap the condensate
  mask — isolating condensate segmentation as the single variable.

FINDING 1 — Elevated dilute density
  The nuclear dilute density for Sample2_5_1 (104.09) is 30-35 units
  higher than every other Sample2_5 entry (range: 68-73). If the dilute
  density matched neighbors (~70), the reference nuclear PC would be
  ~658/70 = 9.4 instead of 6.32. This suggests the reference value
  may be understated, not overstated.

  Possible cause: the manual ImageJ condensate mask for this ROI was
  slightly loose, leaving bright condensate-edge pixels counted as
  dilute-phase background, raising the dilute density and deflating PC.

FINDING 2 — Background subtraction not in automated pipeline
  The person who computed the manual reference CSVs confirmed she applied
  background subtraction before computing PC. The automated pipeline
  does NOT do this. Background subtraction inflates PC because it hits
  the denominator disproportionately:

    PC_bg = (condensate_density - B) / (dilute_density - B)

  Since dilute density is much closer to background than condensate
  density, subtracting B lowers the denominator more (proportionally),
  pushing PC up. This means the reference PC = 6.32 and the pipeline
  PC = 3.33 (Cellpose 3) are NOT computed the same way — part of the
  gap is methodological, not just segmentation quality.

  Need to confirm: what region was used to estimate background B?
  (outside all cells, or something else?)

WHAT PC THE CURRENT PIPELINE COMPUTES
  The automated pipeline computes nuclear PC, raw (no background subtraction):

    cond_pixels   = Ch2[ cond_mask & nuc_mask ]    (inside condensate AND nucleus)
    dilute_pixels = Ch2[ nuc_mask & ~cond_mask ]   (inside nucleus, outside condensates)
    PC = mean(cond_pixels) / mean(dilute_pixels)

  This is equivalent to the nuclear PC column in the reference CSV,
  EXCEPT the reference also applies background subtraction.

TYPES OF PC (for clarity)
  Nuclear PC      : condensate vs. dilute phase, both measured inside the nucleus.
                    What the pipeline computes. Reference = 6.32 (bg-subtracted).
  Cytoplasmic PC  : condensate vs. dilute phase, both measured in the cytoplasm.
                    Reference = 4.84. Pipeline does not currently compute this.
  Bg-subtracted PC: same formula but with a background constant B (measured
                    outside all cells) subtracted from both densities before
                    the ratio. Inflates PC relative to raw. Reference uses this.

  - Add background subtraction to the automated pipeline to match the reference method.
  - Recompute PC with bg subtraction and compare to reference 6.32 directly.

  method from the Fabrini et al. manuscript (section 1.8.4).

REFERENCE PC FORMULA (Fabrini et al., 1.8.4)
  PC = Fluor. density_condensed / Fluor. density_dilute

  Fluor. density_condensed:
    1. B = minimum voxel intensity across the field of view
    2. Mask condensate region (inside condensate ∩ nucleus)
    3. Subtract B from each voxel, clip to 0
    4. Sum all background-subtracted voxel intensities
    5. Divide by total number of voxels in the mask
    -> = Σ clip(pixel - B, 0) / n_voxels_in_mask

  Fluor. density_dilute:
    1. Same B (min voxel intensity across FOV)
    2. Crop a 10×10×10 voxel patch from the dilute phase
       (inside nucleus, outside condensates)
    3. Subtract B, clip to 0
    4. Mean of the patch
    -> = mean clip(pixel - B, 0) over the 10×10×10 patch

IMPLEMENTATION
  Updated cellpose3_segmentation.py Step 4 to match this formula.
  Key changes:
    - B = cond_stack.min() (minimum voxel across full 3D FOV)
    - Condensed density: vectorized over all 3D nuclear condensate voxels
    - Dilute density: searches for a valid 10×10×10 patch entirely within
      the nuclear dilute phase (rng seed=42 for reproducibility);
      falls back to all dilute pixels if no patch found
    - Removed the old slice-by-slice loop for PC; kept Cellpose 3 (do_3D)
      as the segmentation model

DECISION
  Cellpose 3 (cyto3 + DenoiseModel + do_3D=True) stays as the pipeline
  model. The bg-subtraction update is the only PC formula change needed.
  Will rerun to get updated PC vs. reference 6.32.

2026-04-30  Pipeline Summary — How It Works
----------------------------------------------------

PIPELINE STEPS

  [1/6] Load
    Reads two TIF Z-stacks: condensate channel (Ch2) and nuclei channel (Ch1).
    Each stack is 55 x 185 x 259 pixels.

  [2/6] Denoise
    Cellpose 3 DenoiseModel runs on every Z-slice before segmentation.
    Sharpens condensate boundaries without modifying the raw pixel values
    used for intensity measurements.

  [3/6] Segment
    Cellpose 3 (cyto3, do_3D=True) segments both channels in full 3D,
    tracking objects consistently across Z rather than slice-by-slice.
    Post-processing on nuclei: binary mask -> 3D connected components ->
    drop fragments < 1000 voxels, giving 5 clean nuclei from 76 raw labels.

  [4/6] Measure
    regionprops_table extracts area, centroid, and mean intensity per
    object per Z-slice.

  [5/6] 3D Volume
    Counts total voxels per label across all 55 slices.

  [6/6] Partition Coefficient (see below)

PARTITION COEFFICIENT FORMULA

  B            = minimum pixel value across the full 3D stack
                 (removes camera/autofluorescence background offset)

  cond_density = mean( clip(pixel - B, 0) )
                 over all voxels inside BOTH condensate mask AND nucleus mask

  dil_density  = mean( clip(pixel - B, 0) )
                 over the 50 lowest-intensity valid 10x10x10 patches
                 found entirely within the nuclear dilute region

  PC = cond_density / dil_density

  Interpretation: how much brighter are condensates compared to the
  surrounding nuclear background, after removing the camera offset.

THREE IMPROVEMENTS THAT BROUGHT PC FROM ~3.3 TO 6.297

  1. Background subtraction (added April 25)
     The reference was computed with B subtracted from all pixels before
     taking the ratio; the old pipeline was not doing this. Adding it was
     the largest single jump in PC.

  2. Connected-component nuclei relabeling (added April 30)
     Cellpose splits each nucleus into ~15-25 fragments due to internal
     condensate texture. Taking the binary mask (nuclei > 0) and running
     3D connected-component analysis collapses 76 Cellpose labels into
     5 true connected regions. This cleans up which pixels count as
     dilute phase without meaningfully changing the nuclear area covered.

  3. Lowest-intensity patch selection for dilute density (added April 30)
     The old approach picked a single random 10x10x10 patch — PC swung
     from 4.3 to 6.0 depending on the random seed. The fix: find all
     ~88,000 valid patches in the dilute region, sort by mean intensity,
     and average the 50 quietest. This is deterministic, stable, and
     matches how a researcher manually selects a quiet representative
     background spot.

NUCLEUS SELECTION EXPERIMENTS
  Each Cut ROI is nominally a single-cell crop but often contains
  fragments of neighboring cells. Cellpose detects multiple nuclei per
  image. The reference PC was computed per cell (one specific nucleus),
  so pooling all nuclei gives a field-average PC, not per-cell PC.

  Five approaches tested across all 30 JABr cells:

  Approach                           r      RMSE   mean bias
  ─────────────────────────────────────────────────────────────
  All nuclei pooled (original)     0.681   3.250    -13.4%
  Largest nucleus only             0.645   3.298     -9.4%   (worse)
  Central nucleus (closest to      0.716   3.248    -16.0%

  Central nucleus + top-75%        0.735   2.815     +2.8%  <- BEST
  Max-overlap nucleus + top-75%    0.708   3.024     +2.1%

  - Tornado, the only true held-out construct, performs WORSE with
    the trained model (r=0.17) than with cyto3 baseline (r=0.23).
    This is a strict overfitting signature — the model memorized
    in-distribution constructs and degrades on unseen ones.
  - Isotonic calibration is strictly better than linear for the
    trained model output (non-linear pipeline_pc -> ref_pc relationship).
    Mixing in 30% cyto3 PC before isotonic gives a further ~+0.03 lift
    on JABr (where we have cyto3 baseline data). Same approach hurts
    Tornado because both signals are noisy there.
  - Multi-feature ridge (pipeline_pc + cond_density + dilute_density)
    underperforms isotonic on a single feature. Added features just
    inject noise.
  - cellprob_threshold sweeps had marginal effect (mean cond intensity
    moved 1166 -> 1068 between cp=0 and cp=-3). Not the right knob.

WHAT WORKED
  - Per-construct isotonic calibration as the production approach.
    On AABr, calibration lifts raw r=0.15 to r=0.45 — entirely from
    rank-order rebinning. Saved as calibration_table.json, loaded by
    pipeline.py at startup.
  - Ensemble at the PC level (trained + cyto3 weighted mean) helps
    JABr by ~+0.03 r. Needs paired cyto3 baselines which we only have
    for JABr and Tornado.

WHAT DIDN'T WORK
  - cond_topx and cellprob_threshold tuning past the original choices.
  - Linear and polynomial calibration (compared to isotonic).
  - The trained model on held-out data (Tornado, r dropped vs cyto3).
  - Calibration on Tornado specifically — too few cells, too noisy.

ADDITIONAL EXPERIMENTS RUN
  - Optimal ensemble weight sweep on JABr (every w in [0,1] with
    isotonic recalibration each step). Best is w=0.85 trained,
    r=0.903 LOO — confirms JABr ceiling at r ~ 0.90 < 0.92.
  - Diameter sweep on AABr at d in {10,15,20,30}. All explicit
    diameters perform STRICTLY WORSE than Cellpose's auto-detect
    (None) — r drops from baseline 0.148 to between -0.47 and -0.07.
    Conclusion: auto-detect was already correct, diameter is not a
    useful tuning knob here.
  - Spearman-vs-Pearson check across all 5 constructs shows GABr
    (Pearson 0.58 -> Spearman 0.70), AABr (0.15 -> 0.48), and
    JABr_4arm (0.81 -> 0.88) have meaningful rank-order signal that
    Pearson hides via outliers. Tornado is bad on BOTH (Spearman
    0.17), confirming no generalization signal there. This validates
    isotonic as the right calibration family.
  - For Tornado specifically: switched the calibration_table.json
    entry to fit on cyto3 baseline, not trained. cyto3+isotonic gives
    Tornado r=0.13 (vs trained+isotonic r~0.01) — small but the right
    choice; pipeline.py will use this when --construct Tornado is set.
