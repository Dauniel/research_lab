CELLPOSE 3 vs OLD CELLPOSE — KEY DIFFERENCES
  Three concrete changes account for the PC improvement (2.04 → 3.33):

  1. Denoising (new)
     Cellpose 3 runs DenoiseModel cyto3 on every slice before
     segmentation, suppressing shot noise and sharpening condensate
     boundaries. The old pipeline segments the raw image directly
     with no preprocessing. Note: PC intensities are always computed
     from the original raw pixels — denoising only guides boundary
     placement, not the intensity values used in the ratio.

  2. 3D mode (new)
     Cellpose 3 uses do_3D=True, processing XY, XZ, and YZ planes
     simultaneously and merging gradient flows across all three before
     drawing boundaries. The old pipeline runs 2D segmentation
     independently on each of the 55 Z-slices with no cross-slice
     awareness. A condensate spanning 3 slices gets segmented 3
     separate times, often with inconsistent boundaries per slice —
     directly hurting the intensity ratio.

  3. Model version: cyto3 vs cyto2
     cyto3 is trained on a larger and more diverse dataset than cyto2,
     and is generally better at small, dim, or irregularly shaped
     objects. The 3D mode is likely the largest single contributor
     to the PC improvement.

PC COMPARABILITY NOTE
  The PC formula is identical across all models:
    cond_pixels   = Ch2[ cond_mask & nuc_mask ]   (inside condensate AND nucleus)
    dilute_pixels = Ch2[ nuc_mask & ~cond_mask ]  (inside nucleus, outside condensate)
    PC = mean(cond_pixels) / mean(dilute_pixels)

  However, the masks feeding into this formula differ:

  Model            Condensate mask         Nuclei mask
  ---------------  ----------------------  ----------------------
  Old Cellpose     Cellpose cyto2 (2D)     Cellpose cyto2 (2D)
  Cellpose 3       Cellpose cyto3 (3D)     Cellpose cyto3 (3D)
  StarDist         StarDist2D              StarDist2D
  U-FISH           U-FISH spot + disk      Otsu threshold (!)
  Nellie           Frangi + graph          Frangi + graph

  U-FISH is the outlier — it uses a simple Otsu threshold with
  morphological closing for the nuclei mask rather than a trained
  model. This gives it a different nuclear boundary than the others,
  directly affecting which pixels count as dilute-phase and therefore
  the PC. Its high PC (3.81) should be interpreted with this caveat.

  For a truly fair comparison, all models should use the same nuclear
  mask (e.g. Cellpose 3 nuclei masks) and only swap the condensate
  mask — isolating condensate segmentation as the single variable.

FINDING 1 — Elevated dilute density
  The nuclear dilute density for Sample2_5_1 (104.09) is 30-35 units
  higher than every other Sample2_5 entry (range: 68-73). If the dilute
  density matched neighbors (~70), the reference nuclear PC would be
  ~658/70 = 9.4 instead of 6.32. This suggests the reference value
  may be understated, not overstated.

  Possible cause: the manual ImageJ condensate mask for this ROI was
  slightly loose, leaving bright condensate-edge pixels counted as
  dilute-phase background, raising the dilute density and deflating PC.

FINDING 2 — Background subtraction not in automated pipeline
  The person who computed the manual reference CSVs confirmed she applied
  background subtraction before computing PC. The automated pipeline
  does NOT do this. Background subtraction inflates PC because it hits
  the denominator disproportionately:

    PC_bg = (condensate_density - B) / (dilute_density - B)

  Since dilute density is much closer to background than condensate
  density, subtracting B lowers the denominator more (proportionally),
  pushing PC up. This means the reference PC = 6.32 and the pipeline
  PC = 3.33 (Cellpose 3) are NOT computed the same way — part of the
  gap is methodological, not just segmentation quality.

  Need to confirm: what region was used to estimate background B?
  (outside all cells, or something else?)

WHAT PC THE CURRENT PIPELINE COMPUTES
  The automated pipeline computes nuclear PC, raw (no background subtraction):

    cond_pixels   = Ch2[ cond_mask & nuc_mask ]    (inside condensate AND nucleus)
    dilute_pixels = Ch2[ nuc_mask & ~cond_mask ]   (inside nucleus, outside condensates)
    PC = mean(cond_pixels) / mean(dilute_pixels)

  This is equivalent to the nuclear PC column in the reference CSV,
  EXCEPT the reference also applies background subtraction.

TYPES OF PC (for clarity)
  Nuclear PC      : condensate vs. dilute phase, both measured inside the nucleus.
                    What the pipeline computes. Reference = 6.32 (bg-subtracted).
  Cytoplasmic PC  : condensate vs. dilute phase, both measured in the cytoplasm.
                    Reference = 4.84. Pipeline does not currently compute this.
  Bg-subtracted PC: same formula but with a background constant B (measured
                    outside all cells) subtracted from both densities before
                    the ratio. Inflates PC relative to raw. Reference uses this.
